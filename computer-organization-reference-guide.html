Computer Organization Reference Guide

Study workflow: identify the relevant topic, apply the matching pattern, then substitute values. Show method: formula -> substitution -> result. State assumptions for invented ISA questions. For CPU speed, never conclude from GHz alone: T = Ic*CPI/f = Ic*CPI*cycle_time, CPI=sum(CPI_i*I_i)/Ic, MIPS=f/(CPI*10^6). MIPS/MFLOPS not reliable across ISAs; real benchmark time is better.

Digital Logic

Use this when asked: truth table to expression, Boolean laws, NAND/NOR/XOR, or "show gates are functionally complete."

Truth table -> expression: for every row with output 1, write one minterm. Use variable if bit=1, complement if bit=0. OR all minterms. Simplification not required unless asked.

Laws: De Morgan: not(A*B)=not A + not B, not(A+B)=not A * not B. XOR=exactly one 1.

Functional complete: AND+NOT can build OR: A+B = not((not A)*(not B)). Thus AND, OR, NOT are possible.

Radix + Signed

Use this when asked: convert bases, write bit patterns for positive/negative numbers, find range, or interpret a bit pattern.

Value: (...a2 a1 a0.a-1...)_r = sum a_i*r^i.

Decimal -> base r int: repeatedly divide by r; remainders read upward. Fraction: repeatedly multiply by r; integer parts read downward.

Binary/hex: group 4 bits from radix point. Prefer hex for long binary answers.

Sign extension 2's: copy old sign bit into new upper bits.

Floating Point

Use this when asked: encode/decode S|E|f formats, find bias, largest/smallest value, or explain FP add/mul/div. First write the format and bias.

Generic: V=(-1)^S * 1.f * 2^(E-b). Bias for m exponent bits: b=2^(m-1)-1.

Encode decimal x: sign -> convert |x| to binary -> normalize as 1.f * 2^e -> stored exponent E=e+b -> store fraction bits after hidden 1; pad/truncate.

Decode hex: hex -> binary -> split S/E/f -> compute E-b -> evaluate 1.f * 2^(E-b).

No special exponent patternsAnswer
Smallest positive1.0 * 2^(0-b)
Largest positive(2-2^-Fbits) * 2^((2^Ebits-1)-b)
Increase rangeuse more exponent bits, fewer fraction bits -> less precision
If IEEE special patterns are considered, exponent all-0/all-1 are special; valid normalized exponents are 1..2^m-2. Include these details only when explicitly required.

FP add/sub: check zero; align smaller exponent by shifting significand right; add/sub significands; normalize; round. If significand becomes 10.x, shift right 1 and increment exponent.

FP mul: sign=S1 xor S2; E=E1+E2-b; multiply significands; normalize/round; check overflow/underflow.

FP div: sign=S1 xor S2; E=E1-E2+b; divide significands; normalize/round.

Cache Mapping

Use this when asked: set/tag/offset, same-block addresses, direct vs set-associative vs fully associative, or cache final contents.

Always compute in bytes if memory is byte-addressable.

block_bytes = block_words * bytes_per_word

cache_lines = cache_size / block_size

sets = cache_lines / ways

block# = floor(addr/block_bytes), offset=addr mod block_bytes

OrgIndexTag
Directline=block# mod linesfloor(block#/lines)
k-wayset=block# mod setsfloor(block#/sets)
Fully assocnoneblock#

Bits: offset bits=log2(block_bytes). Direct: line bits=log2(lines). k-way: set bits=log2(sets). Tag=remaining upper bits.

Same block addresses: start=addr-offset, end=start+block_bytes-1.

Tracing: for each block: map to set/line; hit if present. Miss if absent. If full, evict FIFO oldest or LRU least recently used. Direct map has no choice.

Cache Timing

Use this when asked: hit rate, miss rate, miss penalty, AMAT, or multi-level cache timing. Keep units as ns/ms.

AMAT = hit_time + miss_rate * miss_penalty

hit_rate = 1 - miss_rate

Course trace style: if each listed block has avg R references, total_refs = listed_blocks*R, hit_rate = 1 - misses/total_refs.

Penalty if one word first then later words: first + (block_words-1)*later. If memory transfers g words at a time: groups=ceil(block_words/g), penalty=first_group+(groups-1)*later_group.

Multi-level: L1 + MR1*(L2 + MR2*(L3 + MR3*Mem)).

Write-through: write cache and memory; consistent but slower/needs buffer.

Write-back: write memory only on replacement if dirty; faster but memory inconsistent.

Locality: temporal = recently used likely reused. Spatial = nearby addresses likely used. Row-major C arrays: make last subscript the inner loop.

Split cache: instruction cache + data cache. Reduces structural/resource hazard between instruction fetch and data access; does not solve data/control hazards.

Disk, RAID, SSD

Use this when asked: rpm, average rotational latency, sector timing, RAID reliability/capacity, or whether SSD is suitable for page file.

Disk access = seek + rotational delay + transfer.

rev_time_ms = 60000/rpm; average rotational latency=rev_time/2 = 30000/rpm.

sector_time = rev_time / sectors_per_track.

Consecutive n sectors same track: seek + avg_latency + n*sector_time. Scattered n sectors: n*(seek + avg_latency + sector_time).

RAIDKey point
0striping, no redundancy, fastest, one disk fail loses data
1mirroring, high reliability, reads can use either copy, writes duplicated
3/4single dedicated parity disk; recover 1 failure; parity bottleneck
5distributed parity; survives 1 disk failure; capacity=(N-1)*disk
6two distributed parity values; survives 2 disk failures; capacity=(N-2)*disk
SSD for page file: suitable vs HDD due low access time and random I/O, but still much slower than RAM; heavy writes cause wear; more RAM is better.

Virtual Memory

Use this when asked: page fault, TLB, logical vs physical address, demand paging, or page file.
Logical/virtual address is program-visible; physical address accesses RAM. MMU maps virtual page number -> physical frame number; offset copied unchanged. PTE contains valid bit V, protection P, dirty D, physical frame number. Page fault: locate page on disk -> find/free/evict frame -> if victim dirty write back -> invalidate old PTE -> load page -> update PTE -> restart process. TLB caches page-table entries.

Pipeline + RISC

Use this when asked: identify hazards, explain split cache, or compare RISC/simple instruction design.

Stages: FI, DI, CO, FO, EI, WO. Modern 5-stage often IF, ID, EX, MEM, WB. Ideal pipeline: each instruction still takes several stages but throughput is 1 instruction/cycle, CPI about 1.

Structural/resource hazard: same hardware needed by two stages; fix by more resources: separate I/D cache, PC incrementer, more buses.

Data hazard: later instruction needs previous result. Main simple pipeline hazard is RAW. Fix by scheduling independent instructions, stalls, or forwarding.

Control hazard: branch/CALL changes PC; next fetch unknown until branch resolved. Branch prediction reduces stalls.

Worked hazard example:
ADD R1,R2,R3
SUB R3,R4,R5
BMI L1
SUB has RAW data hazard because it reads R3 produced by ADD.
BMI depends on SUB flag/result and also causes control hazard because PC may change.
RISC: register-register ALU ops; only LOAD/STORE access memory; fixed/simple formats; few modes; hardwired control; pipelining. Tradeoff: maybe more instructions, but lower CPI and clock cycle time.

Short Concepts

Use these when asked: short 2-3 mark definitions. Include one precise sentence plus one reason/example.

Direct vs indirect: direct instruction contains operand address; indirect contains address of a memory cell that contains the real operand address.

FIFO: replace block that entered cache earliest. LRU: replace block unused for longest time.

Memory hierarchy downwards: cost/bit decreases, capacity increases, access time increases, CPU access frequency decreases.

SRAM vs DRAM: SRAM uses logic gates/latches, no refresh, faster, cache. DRAM uses charge, needs refresh, slower, main memory.

Endian: big endian stores most significant byte at lowest address; little endian stores least significant byte at lowest address.

Memory-mapped I/O: I/O registers placed in memory address space; normal load/store read/write CSR/data registers.

Page / TLB Answers

Use this when asked: why virtual memory is useful, what a page fault is, or why TLB helps performance.

Why virtual memory: program sees large logical address space; physical RAM may be smaller; OS/MMU manage movement and protection.

Demand paging: pages loaded only when needed; fast startup; early page faults until working set loaded.

Page fault answer: page not in RAM; OS finds page on disk, finds/free frame, writes dirty victim if needed, updates page tables, restarts faulting instruction/process later.

TLB: cache of page table entries; avoids one extra memory access for page-table lookup on most references.

Answer Phrases

Use these when asked: conceptual "explain why" questions. Select the relevant idea and add one course keyword.

Programmed I/O bad: CPU repeatedly checks status in a loop and wastes cycles while device is slow.

Interrupt advantage: CPU can execute other code until device signals completion.

DMA advantage: large block transfer directly between I/O controller and memory with little CPU involvement.

Split cache hazard: avoids structural conflict when pipeline fetches instruction and accesses data in same cycle.

Direct map pro/con: simple and fast; conflict misses if active blocks map to same line.

Fully assoc pro/con: flexible and higher hit ratio; must compare many tags.

Set assoc: compromise between direct and fully associative.

Floating-point caveat: fixed total bit patterns, so more range means less precision; numbers are not evenly spaced.

Arithmetic Extras

Use this when asked: relative error, multiplication proof idea, or product bit bound. These are lower-frequency but cheap marks if they appear.

Relative error: if exact value is A and stored value is A', |A-A'|/|A| (signed form only if asked). For non-terminating binary fractions, write enough repeating bits, truncate/round, reconstruct A', then compute.

2's multiplication idea: sign-extend partial products to double precision; for multiplier sign bit 1 add two's complement of multiplicand; ignore carry out. Avoid proof unless asked.

Unsigned product bound: n-bit max times m-bit max is (2^n-1)(2^m-1) < 2^(n+m), so product needs at most n+m bits.

OS / Protection

Use this when asked: user/kernel mode, system calls, protected resources, or time sharing.
User mode cannot directly use protected resources or switch itself to kernel mode. System calls enter OS at controlled entry points and run in kernel/supervisor mode. Time sharing: when a process blocks on I/O or uses up its time slice, OS schedules another process.

Spatial locality code: in row-major arrays, use for i rows outside and for j cols inside so a[i][j] touches consecutive memory.

RAID 5 capacity: with N equal disks, usable capacity=(N-1)*disk. Example six 10T disks -> 50T usable.

VM Address Calc

Use this when asked: translate a virtual/logical address using a page table or TLB.
page_size = 2^p bytes -> offset has p bits
VPN = floor(VA/page_size)
offset = VA mod page_size
If PTE valid: PA = PFN*page_size + offset
If PTE invalid: page fault; OS loads page, updates PTE/TLB, restarts.
