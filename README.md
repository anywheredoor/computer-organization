# Computer Organization

A compact two-page reference guide for core computer organization concepts. It was originally created for a final examination where one printed, double-sided A4 reference sheet was allowed, so the material is condensed into formulas, worked examples, and implementation patterns.

## Reference Guide

- [PDF reference guide](computer-organization-reference-guide.pdf)
- [HTML source used to generate the PDF](computer-organization-reference-guide.html)

## Topics Covered

- Digital logic and Boolean expressions
- Radix conversion, signed number representations, and floating point
- Cache mapping, cache timing, RAID, SSDs, and virtual memory
- Addressing modes, instruction formats, and datapath micro-operations
- Pipeline hazards, RISC design, programmed I/O, DMA, and interrupts
- Worked examples for common numeric and architectural calculations

## What I Learned

Through this project, I strengthened my understanding of how computer systems represent data, move information through memory hierarchies, and execute instructions at the architectural level.

- Data representation: Boolean logic, radix conversion, signed integer formats, and floating-point encoding
- Memory systems: cache mapping, hit/miss behavior, AMAT, write policies, locality, RAID, SSDs, and virtual memory
- Processor organization: addressing modes, instruction encoding, datapath micro-operations, stack behavior, and subroutine control flow
- Performance analysis: instruction count, CPI, clock rate, pipeline hazards, RISC tradeoffs, and memory-system effects
- I/O and operating systems: programmed I/O, interrupts, DMA, device status registers, protection, and time sharing
- Technical communication: condensing dense systems material into a structured reference with clear formulas, examples, and decision rules
